# 发布说明与加载方式

## 本版定位

这是 HGS 的正式发布结构，不再使用“单入口大文件”或“多入口并存”的方式。

## 推荐仓库布局

```text
HGS_Skill/
├─ release/
│  ├─ MANIFEST.json
│  ├─ 00_HGS_Master_Loader.md
│  ├─ roles/
│  ├─ protocols/
│  └─ docs/
```

## 推荐启动口令

```text
读取 MANIFEST.json，并按 load_order 装配整个 HGS 正式发布版。
本轮只承认 release/ 下文件为有效 Skill 源。
执行模式：full_loop。
我后续上传的文件默认进入：
P10 初判 → P9 审查派单 → P8 执行 → 体验验证 → P9 复审 → P10 终审（按需） → Closeout。
命中 risk_gates 时显式停下。
```

## 不推荐做法

- 只读 Master Loader，不读 roles/ 和 protocols/
- 在仓库里继续堆旧版兼容层
- 在角色文件内各自维护不同的 Exec Report / Review 格式
- 用 README 代替 Manifest
