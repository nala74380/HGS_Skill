# 三、当前通过的核心原因

## 1. 角色总表已升级到 r2
现在角色文档已经不只写：
- 谁说了算
- 谁不能越界

还写清了：
- 默认先跑哪些工具
- 默认命中哪些门禁动作
- 必须落哪些协议字段
- 清零循环里承担什么责任

## 2. 工具绑定关系已显式化
通过《角色-工具矩阵总表》，角色与工具的默认绑定关系已显式治理，不再靠隐含理解。

## 3. 门禁动作已 active 化
高风险、越权、运行面门禁已进入 active chain，边界不再只是文档原则，而是变成了执行规则。

---

# 四、为什么还有 3 项是 conditional pass

这 3 项不是因为静态治理不清，而是因为它们更依赖真实复杂场景验证：

1. **Control Plane vs Execution Plane**  
   在复杂运行面批次下，仍建议继续观察后台真相与运行事实的交叉场景。

2. **SRE vs Execution Plane**  
   incident replay 与 runtime gate 已接入，但仍建议继续观察真实复杂事故复盘时的边界稳定性。

3. **Docs vs Unresolved Truth**  
   116 已能巡检文档状态漂移，但复杂多轮升级中仍建议继续观察文档是否会早于真相裁定收敛。
