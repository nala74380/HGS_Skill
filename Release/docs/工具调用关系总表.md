# HGS 工具调用关系总表

> 版本：formal-2026-03-31-r2  
> 目的：把 HGS 当前已经落库并已接入装配链的 Tool Skill 的调用规则、优先角色、适用场景、结果落点与越界边界一次性钉死。  
> 说明：本文件是**工具治理文档**，不是运行入口；不替代 `MANIFEST` / `Master Loader`，但为正式装配链提供唯一工具调用依据。

# 一、总原则

## 1. 工具不是角色

HGS 中的工具 Skill 只做四件事：

1. 结构化分析  
2. 证据抽取  
3. 差异识别  
4. 结果格式化  

工具 **不能** 做四件事：

1. 替代 Owner 拍板真相  
2. 越权定义业务规则  
3. 宣布已闭环  
4. 绕过 P9 / P10 的裁决链  

一句话：

**工具负责“看清楚”，角色负责“拍板、执行、验证、收口”。**

## 2. 工具调用的基本顺序

```text
先识别真相 Owner / 执行 Owner
  ↓
再选对应工具做结构化分析
  ↓
工具结果进入 issue evidence / verdict / plan
  ↓
角色基于工具结果拍板或执行
```

## 3. 工具结果的默认落点

工具输出不得悬空，必须进入至少一个协议或角色输出：

- `ISSUE-LEDGER`
- `P8-EXEC-REPORT`
- `P9-REVIEW-VERDICT`
- `QA-VALIDATION-PLAN`
- `QA-VERIFICATION-RESULT`
- `BUSINESS-RULE-DECISION`
- `IDENTITY-DECISION`
- `BILLING-VERDICT`
- `RELEASE-CONFIG-VERDICT`
- `SECURITY-RISK-VERDICT`
- `CONTROL-PLANE-VERDICT`
- `EXECUTION-PLANE-VERDICT`
- `AGENT-OPS-VERDICT`
- `ENDUSER-SUPPORT-VERDICT`
- `DOCS-KNOWLEDGE-UPDATE`
- `HGS-CLOSEOUT`

# 七、哪些场景必须至少调用一个工具

以下场景禁止纯靠口头判断，必须至少调用一个 Tool Skill：

1. **规则冲突**：至少调用 `70` 或 `71`  
2. **身份 / token / 刷新链争议**：至少调用 `72` / `74` / `88` / `90` 之一  
3. **名额 / 配额 / 点数 / 冻结争议**：至少调用 `76` / `77` / `78` 之一  
4. **接口联调争议**：至少调用 `79` 或 `82`  
5. **版本兼容争议**：至少调用 `101`  
6. **宣称已验证**：至少调用 `95` 或 `96` 之一  
7. **要沉淀成 SOP**：至少调用 `106`  
8. **协议字段缺失 / evidence 悬空 / closeout 断链**：必须调用 `107`  
9. **自动分流不清 / 怀疑错派 / 漏工具 / 漏验证**：必须调用 `108`  
10. **高风险动作门禁争议**：必须调用 `109`  
11. **越权 / 绕过路径争议**：必须调用 `110`

# 八、工具越界红线

## 1. 不能替代真相角色

- `70` 不能替代 Product 拍板规则  
- `72/73/74/88/89/90` 不能替代 Auth / Control Plane / Console Runtime 拍板真相  
- `76/77/78` 不能替代 Billing 拍板账务真相  
- `101` 不能替代 Release / P10 拍板是否接受发布风险  
- `109/110` 不能替代 Security / Risk Owner 或 P10 拍板最终风险容忍度  

## 2. 不能替代验证结论

- `95` 不能等于“已经验证通过”  
- `96` 不能等于“已经全部测完”  
- `107` 不能等于“字段齐了就代表问题解决了”  
- `108` 不能等于“模拟走通就代表真实执行没问题”  

## 3. 不能替代文档治理

- `106` 不能把未拍板争议写进 SOP  
- `106` 不能替代 Docs Owner 做适用范围和受众把关  

## 4. 不能绕过 P9

- 工具发现多 owner 冲突、contract 漂移、状态漂移、错派或绕过风险时，必须回到 P9 拆单或复审，不得直接各修各的。

# 十、结论

到当前阶段，HGS 工具体系已经从“散装分析器”升级为：

- 真相判定工具
- 差异 / 漂移工具
- 体验审查工具
- 验证与治理工具
- 安全 / 门禁工具
- 沉淀工具

**让每个角色知道什么时候该先用哪个工具，工具结果该落到哪，工具不能替代谁。**
